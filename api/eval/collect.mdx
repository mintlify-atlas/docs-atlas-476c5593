The `pipeline.eval.collect` module provides the main orchestration functions for collecting evaluations across multiple models and scenarios.

## collect_core_evaluations

Collect one scenario's criterion-wise evaluations using configurable sampling strategies.

```python
collect_core_evaluations(
    criteria,
    scenario,
    scenario_index,
    models,
    evaluations,
    sampler_mode="random_judge_group",
    allow_ties=True,
    group_size=4,
    groups=1,
    alpha=2.0,
    cached_responses_by_scenario=None,
    judge_prompt_prefix_fn=None,
    max_tokens=4096,
    verbose=False,
)
```

### Parameters

<ParamField path="criteria" type="list" required>
  List of evaluation criteria to assess models against
</ParamField>

<ParamField path="scenario" type="str" required>
  The scenario text to evaluate models on
</ParamField>

<ParamField path="scenario_index" type="int" required>
  Index of the scenario in the evaluation dataset
</ParamField>

<ParamField path="models" type="dict" required>
  Dictionary mapping model nicknames to their full model names
</ParamField>

<ParamField path="evaluations" type="list" required>
  List of existing evaluations for this scenario
</ParamField>

<ParamField path="sampler_mode" type="str" default="random_judge_group">
  Sampling strategy for judge and evaluee selection:
  - `random_judge_group`: Recommended default, selects random judge and one random group of evaluees
  - `adaptive_inverse_count`: Balances under-sampled judges and evaluees
  - `uniform`: Baseline uniform random sampling
</ParamField>

<ParamField path="allow_ties" type="bool" default="True">
  Whether to allow tie results in evaluations
</ParamField>

<ParamField path="group_size" type="int" default="4">
  Number of evaluees judged together in each sampled group. Recommended default is 4 for most populations.
</ParamField>

<ParamField path="groups" type="int" default="1">
  Number of sampled judge+group batches to run for this scenario. Increase this before increasing group_size for better coverage.
</ParamField>

<ParamField path="alpha" type="float" default="2.0">
  In adaptive inverse-count sampling, larger alpha increases preference for under-sampled judges/evaluees. alpha=0 is uniform. Practical range is usually 1.0-2.0.
</ParamField>

<ParamField path="cached_responses_by_scenario" type="dict | None" default="None">
  Optional cache of model responses keyed by scenario index to avoid redundant API calls
</ParamField>

<ParamField path="judge_prompt_prefix_fn" type="callable | None" default="None">
  Optional function to generate custom prefix for judge prompts
</ParamField>

<ParamField path="max_tokens" type="int" default="4096">
  Maximum tokens for model responses
</ParamField>

<ParamField path="verbose" type="bool" default="False">
  Enable verbose logging of evaluation progress
</ParamField>

### Returns

<ResponseField name="new_evaluations" type="list">
  List of new evaluation results collected for this scenario
</ResponseField>

### Example

```python
from pipeline.eval.collect import collect_core_evaluations

criteria = [
    "Helpfulness",
    "Accuracy",
    "Clarity"
]

models = {
    "gpt4": "gpt-4-turbo-preview",
    "claude": "claude-3-opus-20240229",
    "gemini": "gemini-pro",
    "mixtral": "mixtral-8x7b-instruct"
}

scenario = "Explain the concept of photosynthesis to a 10-year-old."

evaluations = collect_core_evaluations(
    criteria=criteria,
    scenario=scenario,
    scenario_index=0,
    models=models,
    evaluations=[],
    sampler_mode="random_judge_group",
    group_size=4,
    groups=2,
    verbose=True
)
```

## build_judge_and_eval_counts

Build frequency counts for how often each model appears as judge or evaluee.

```python
build_judge_and_eval_counts(comparisons, num_models)
```

### Parameters

<ParamField path="comparisons" type="list" required>
  List of comparison tuples extracted from evaluations
</ParamField>

<ParamField path="num_models" type="int" required>
  Total number of models in the evaluation
</ParamField>

### Returns

<ResponseField name="judge_counts" type="list">
  List of counts for how many times each model served as judge
</ResponseField>

<ResponseField name="eval_counts" type="list">
  List of counts for how many times each model was evaluated
</ResponseField>

### Example

```python
from pipeline.eval.collect import build_judge_and_eval_counts
from pipeline.utils import extract_comparisons_with_ties_criteria

comparisons, _ = extract_comparisons_with_ties_criteria(
    evaluations,
    num_criteria=3,
    verbose=False
)

judge_counts, eval_counts = build_judge_and_eval_counts(
    comparisons,
    num_models=4
)

print(f"Judge counts: {judge_counts}")
print(f"Eval counts: {eval_counts}")
```
