This module provides PyTorch Dataset classes and training functions for BT/BTD models.

## Comparisons

Dataset wrapper for legacy comparison format.

```python
from pipeline.train.train import Comparisons
from torch.utils.data import DataLoader

# Format: [l, i, j, k, r]
# l=scenario, i=judge, j=model1, k=model2, r=result
comparisons = [
    [0, 0, 1, 2, 1.0],  # judge 0 prefers model 1 over 2
    [0, 0, 1, 3, 0.0],  # judge 0 prefers model 3 over 1
]

dataset = Comparisons(comparisons)
loader = DataLoader(dataset, batch_size=32, shuffle=True)
```

### Constructor

<ParamField path="comparisons" type="List[List]" required>
  List of comparisons in format `[l, i, j, k, r]` where:
  - `l`: scenario/prompt index
  - `i`: judge/annotator index
  - `j`: first model index
  - `k`: second model index
  - `r`: result (1.0 if j>k, 0.0 if k>j, 0.5 if tie)
</ParamField>

### \_\_getitem\_\_

Returns a single comparison as tensors.

```python
i, j, k, r = dataset[0]
```

<ParamField path="idx" type="int" required>
  Index of the comparison
</ParamField>

<ResponseField name="output" type="Tuple[Tensor, Tensor, Tensor, Tensor]">
  Tuple of (judge_idx, model1_idx, model2_idx, result) as tensors
</ResponseField>

## CriteriaComparisons

Dataset wrapper for criterion-conditioned comparisons.

```python
from pipeline.train.train import CriteriaComparisons

# Format: [c, l, i, j, k, r]
comparisons = [
    [0, 0, 0, 1, 2, 1.0],  # criterion 0, judge 0 prefers model 1
    [1, 0, 0, 1, 2, 0.0],  # criterion 1, judge 0 prefers model 2
]

dataset = CriteriaComparisons(comparisons)
```

### Constructor

<ParamField path="comparisons" type="List[List]" required>
  List of comparisons in format `[c, l, i, j, k, r]` where:
  - `c`: criterion index
  - `l`: scenario/prompt index
  - `i`: judge/annotator index
  - `j`: first model index
  - `k`: second model index
  - `r`: result (0=tie, 1=j>k, 2=k>j for BTD models)
</ParamField>

### \_\_getitem\_\_

Returns a single comparison as tensors.

```python
c, i, j, k, r = dataset[0]
```

<ParamField path="idx" type="int" required>
  Index of the comparison
</ParamField>

<ResponseField name="output" type="Tuple[Tensor, Tensor, Tensor, Tensor, Tensor]">
  Tuple of (criterion_idx, judge_idx, model1_idx, model2_idx, result) as tensors
</ResponseField>

## train_vector_bt

Main training loop for all BT/BTD model variants.

```python
from pipeline.train.train import train_vector_bt
from pipeline.train.bt_models import VectorBTD
from torch.utils.data import DataLoader

model = VectorBTD(num_models=10, d=32)
loader = DataLoader(dataset, batch_size=32, shuffle=True)

loss_history = train_vector_bt(
    model=model,
    dataloader=loader,
    lr=0.001,
    weight_decay=0.01,
    max_epochs=100,
    device="cuda",
    save_path="./checkpoints",
    use_btd=True,
)
```

<ParamField path="model" type="nn.Module" required>
  BT or BTD model instance to train
</ParamField>

<ParamField path="dataloader" type="DataLoader" required>
  PyTorch DataLoader with comparison data
</ParamField>

<ParamField path="lr" type="float" required>
  Learning rate for Adam optimizer
</ParamField>

<ParamField path="weight_decay" type="float" required>
  L2 regularization weight decay
</ParamField>

<ParamField path="max_epochs" type="int" required>
  Maximum number of training epochs
</ParamField>

<ParamField path="device" type="str" required>
  Device to train on ("cuda" or "cpu")
</ParamField>

<ParamField path="save_path" type="str" default="None">
  Directory to save model checkpoint and loss plot. If None, no saving occurs.
</ParamField>

<ParamField path="normalize" type="bool" default="False">
  Whether to normalize v embeddings after each batch
</ParamField>

<ParamField path="use_btd" type="bool" default="False">
  Whether model is BTD (uses CrossEntropyLoss) vs BT (uses BCELoss)
</ParamField>

<ParamField path="criterion_mode" type="bool" default="False">
  Whether using criterion-conditioned model (CriteriaVectorBTD)
</ParamField>

<ParamField path="verbose" type="bool" default="False">
  Whether to print detailed progress information
</ParamField>

<ResponseField name="output" type="List[float]">
  List of average loss values per epoch
</ResponseField>

<Note>
  Training automatically stops early if loss converges (average change < 0.0001 over last 10 epochs).
</Note>

## group_split_comparisons

Grouped train/test split to minimize data leakage.

```python
from pipeline.train.train import group_split_comparisons

train_comps, test_comps = group_split_comparisons(
    comparisons=all_comparisons,
    test_size=0.2,
    random_state=42,
    verbose=True,
)
```

Groups comparisons by `(scenario, judge, {model1, model2})` and keeps each group entirely in train or test. This prevents the same comparison context from appearing in both splits.

<ParamField path="comparisons" type="List[List]" required>
  List of comparisons in format `[c, l, i, j, k, r]`
</ParamField>

<ParamField path="test_size" type="float" default="0.2">
  Fraction of groups to use for test set (0.0 to 1.0)
</ParamField>

<ParamField path="random_state" type="int" default="42">
  Random seed for reproducible splits. Set to None for random splitting.
</ParamField>

<ParamField path="verbose" type="bool" default="False">
  Whether to print split statistics
</ParamField>

<ResponseField name="output" type="Tuple[List, List]">
  Tuple of (train_comparisons, test_comparisons)
</ResponseField>
